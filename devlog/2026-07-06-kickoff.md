# Devlog 001 — Kickoff: why HomeFleet, and what the research said

**2026-07-06**

## The itch

I have two computers and a local-model habit. Every time my agent grinds through a repo-exploration task or a test suite, the other machine sits idle. The obvious question: why can't my agent just *use* it?

## The fork that shaped everything

"Use my other computer" means two completely different things:

1. **Distributed inference** — pool GPUs/RAM so machines run *one bigger model together*. One brain, two bodies.
2. **Agent delegation** — each machine runs its own model; the agent on machine A hands *whole subtasks* to machine B. Two brains.

Research (five parallel deep-dives, 2026-07-04) settled it. Distributed inference is crowded — exo (45k+ stars), GPUStack, LocalAI federated, llama.cpp RPC — and on a home LAN it's a memory-pooling win, not a speed win. Agent delegation is where the whitespace lives: **nothing** today is a thin daemon + MCP server that makes your other machines appear as `delegate_task` tools inside a stock agent session. The neighbors (Fusion, AgentsMesh, ai-maestro) are platforms — dashboards, kanbans, new protocols. Nobody does LAN auto-discovery. Nobody defaults to local-model executors. GitHub validated the UX pattern itself (their MCP server delegates to Copilot's coding agent) — but cloud-only.

So: HomeFleet distributes tasks, not tensors (ADR-0001), fronted by MCP (ADR-0002).

## Findings worth stealing

- **Timing luck on MCP:** the largest spec revision since launch lands July 28, 2026 — sessions removed, Sampling/Roots/Logging deprecated. Designing stateless from day one turns a rewrite into a codemod.
- **Small models can actually do this now.** Docker's 3,570-test eval: Qwen3 14B hits 0.971 tool-call accuracy (GPT-4 class), 8B holds 0.933 — quantization barely hurts. The floor for reliable agentic behavior is ~7–9B tool-trained models, and context window matters more than parameters: 4k defaults silently lobotomize agents. Enforce ≥16k.
- **The AMD situation resolved itself.** My RX 6700 XT (gfx1031) still has no official Windows ROCm — but Vulkan went mainstream (Ollama ships it by default now) and performs ~equal on RDNA2. The wild part: MoE models with CPU-offload mean this 12GB card + 32GB RAM can run **Qwen3.6-35B-A3B** at ~20–30 tok/s. A 35B-class agentic model on a mid-range 2021 GPU.
- **A GTX 1060 is still useful in 2026** — not for inference (Pascal, 6GB, needs CUDA 12.4 builds since 13.x dropped it) but as an *execution node*: run the tests, compile, grep. Capability roles make weak machines earn their keep.
- **Consumer P2P security is a solved pattern:** Syncthing's device-ID-is-the-cert-fingerprint model — mTLS, fingerprint pinning, pairing codes, no CA, no accounts (ADR-0004). Counter-example: Ollama still ships zero auth; 14k+ instances exposed on Shodan.
- **The executor loop is commodity; the harness isn't.** pi proved a competitive coding agent is 4 tools + a ~1k-token prompt + a while-loop. What mature CLIs actually sell is robust *edit application* — which read-only recon doesn't need. Hence: build the minimal loop (ADR-0003), wrap something like opencode later when code-writing delegation arrives.

## Decisions locked

TypeScript monorepo (pnpm) · Apache-2.0 · MCP front door, stateless, Streamable HTTP on localhost · custom minimal agent executor · Syncthing-style trust · git-bundle workspace sync (ADR-0005) · recon + command execution as the v0.1 wedge · everything testable multi-node on a single machine.
